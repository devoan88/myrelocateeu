# RelocateEU

AI-powered relocation guides for moving to **Austria**, **Germany**, and **Switzerland**. Built with Next.js 14, Supabase, Stripe, and Claude RAG over verified official sources.

## Features

- Multilingual UI (English, German, Georgian)
- Step-by-step checklists with verified content and “last verified” dates
- AI chat and personalized guides (Premium / Pro)
- Stripe subscriptions (Free / Premium €12 / Pro €29)
- Admin dashboard at `/admin` for content review and edits
- Weekly cron job to monitor official source URLs
- GDPR-oriented privacy policy, cookie consent, and user reports for outdated info

## Prerequisites

- Node.js 18+
- [Supabase](https://supabase.com) project with migrations applied
- [Stripe](https://stripe.com) account (test or live)
- [Anthropic](https://anthropic.com) API key
- [Resend](https://resend.com) for admin emails (optional in dev)
- [Vercel](https://vercel.com) for hosting (recommended)

## Local development

1. **Clone and install**

   ```bash
   npm install
   ```

2. **Environment**

   ```bash
   cp .env.example .env.local
   ```

   Fill in at least `NEXT_PUBLIC_SUPABASE_*`, `SUPABASE_SERVICE_ROLE_KEY`, and `NEXT_PUBLIC_APP_URL`. Other keys unlock AI, payments, email, and analytics.

3. **Supabase**

   Run migrations in `supabase/migrations/` (via Supabase CLI or SQL editor), then optionally:

   ```bash
   # If using Supabase CLI
   supabase db push
   ```

4. **Run**

   ```bash
   npm run dev
   ```

   Open [http://localhost:3000](http://localhost:3000).

## Deploy on Vercel

### 1. Import project

1. Push this repo to GitHub/GitLab/Bitbucket.
2. In [Vercel](https://vercel.com/new), import the repository.
3. Framework preset: **Next.js** (detected automatically via `vercel.json`).

### 2. Environment variables

In **Project → Settings → Environment Variables**, add every key from `.env.example` for **Production** (and Preview if needed).

Important:

- Set `NEXT_PUBLIC_APP_URL` to `https://yourdomain.com`
- Use **live** Stripe keys and webhook URL in production
- Set a strong `ADMIN_PASSWORD`
- Set `CRON_SECRET` (Vercel can also provide this for cron jobs)

### 3. Cron job

[`vercel.json`](vercel.json) registers a weekly cron:

- **Path:** `/api/cron/check-updates`
- **Schedule:** Mondays 09:00 UTC (`0 9 * * 1`)

Requires **Vercel Pro** (or compatible plan) for cron. The route checks `Authorization: Bearer <CRON_SECRET>`.

### 4. Stripe webhook

After deploy, add a webhook endpoint in Stripe:

```
https://yourdomain.com/api/webhook
```

Events: `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`

Copy the signing secret to `STRIPE_WEBHOOK_SECRET` in Vercel.

### 5. Custom domain & Cloudflare

See [`docs/CLOUDFLARE_DNS.md`](docs/CLOUDFLARE_DNS.md) for DNS, SSL, and Clerk/auth notes.

### 6. Google Analytics

1. Create a GA4 property and copy the Measurement ID (`G-XXXXXXXXXX`).
2. Set `NEXT_PUBLIC_GA_ID` in Vercel.
3. Analytics loads only after users **accept** cookies (see cookie banner).

### 7. SEO

- **Sitemap:** `https://yourdomain.com/sitemap.xml` (generated by `src/app/sitemap.ts`)
- Ensure `NEXT_PUBLIC_APP_URL` matches your production domain so URLs are correct.

## Security headers

Configured in [`next.config.mjs`](next.config.mjs):

- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy` (restricts camera, microphone, geolocation)

## Admin

- **URL:** `/admin` (redirects to `/admin/login` if not authenticated)
- **Auth:** `ADMIN_PASSWORD` (HTTP-only cookie)
- Edit guide entries; saving sets **last verified** to today
- Review pending updates from the weekly monitor
